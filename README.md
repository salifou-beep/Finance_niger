# Finance_niger
C'est un projet SQL complet qui illustre ma capacité à concevoir, implementer, et analyser une base de données relationnelle complexe pour système financier, en utilsant SQL (DDL,DML). J'ai modélisé les entités fondamentales (Banques, Clients, Comptes, Transactions) et généré un jeu de données fictif ancré dans un économique réél(villes nigériennes, monnaie FCFA) pour garantir la pertinence.
Ojectifs et Compétence Mises en oeuvre:
. Modélisation E-R: Conception d'un schéma relationnel respectant les contraintes d'intégrité via des clés Primaires et Etrangères.
. SQL Avancé : Maîtrise des requêtes Analytiques utilisant les JOINS(inner,multi-table), les fonctions d'Agrégation (SUM, AVG, COUNT), et les conditions de filtrage complexes(WHERE, HAVING).
. Analyse de cohérence : utilisation de Sous-Requêtes pour des analyses précises (ex: identifier les clients ayant des transactions spécifique ou le compte au solde maximum).
. Pertinence Business : Capacité à transformer des besoins métiers (calcul du solde total par banque, identification des clients multicomptes ) en requêtes performantes.
